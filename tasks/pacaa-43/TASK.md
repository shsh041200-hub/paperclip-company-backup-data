## 배경

[PACAA-42](/PACAA/issues/PACAA-42) 보드 요청: 텔레그램에서 CEO에게 이슈 생성 가능한 `/tell_ceo` 명령을 `/help` 메뉴 아래 추가.

## 스코프

1. **조사 (먼저, 짧게):**
   * Paperclip 코드베이스에 텔레그램 봇/통합이 이미 있는지 확인 (`paperclip-dev` 스킬 사용)
   * 있으면: `/help` 위치와 명령 등록 방식 파악
   * 없으면: 최소 봇 (플러그인 또는 사이드카) 옵션 정리
2. **플랜 (보드 컨펌 게이트):**
   * 이 이슈의 `plan` 문서에 최소 설계안 작성:
     * 어디에 코드를 둘지 (플러그인 / 코어 / 외부 봇)
     * 인증 방식 (founder 텔레그램 chat\_id 화이트리스트 권장)
     * `/tell_ceo <title>` 입력 → Packlinx CEO 에이전트 (`e33ecade-45dc-47ea-9d46-78ef72e8831c`) 에게 이슈 자동 할당
     * `/help` 출력에 항목 추가
   * 플랜 작성 후 `request_confirmation` 인터랙션 생성, 보드(이용자 founder) 승인 대기
   * 승인 후 구현 착수
3. **구현:**
   * 최소 기능부터 (title-only 이슈 생성)
   * 봇 토큰은 환경변수, 절대 커밋 금지
   * 부모 [PACAA-42](/PACAA/issues/PACAA-42) 에 진행 상황 코멘트로 동기화

## 제약

* 이건 사이드 프로젝트입니다. 우선순위 medium. 1순위 (제품 데이터/검색) 작업 진행 중이면 간섭 없게 시간 배분하세요.
* 사용자 입력은 외부 신뢰 경계입니다 — 인젝션 방지, chat\_id 화이트리스트 필수.
* 분기 결정 (코어 수정 vs 플러그인 vs 외부 봇) 은 플랜에서 벤치마크와 함께 정당화하세요.

## 완료 기준

* 보드가 실제로 텔레그램 `/tell_ceo`로 이슈를 만들고 그게 CEO 인박스에 도착함을 확인.
* 사용법이 `/help` 응답에 노출됨.
