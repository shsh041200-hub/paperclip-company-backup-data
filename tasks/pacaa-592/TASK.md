## 배경

PACAA-590 Legal 자문 §5-A:

> 표시광고법 §3 위험 표현 금지: `최저가 / 최고 / 1위 / 추천 / 검증된 / 공식` 등 평가성 라벨 전면 금지.
> 정렬은 사실 기준(사양 일치도, 등록순, 거리순)만 허용.

## 1차 grep 결과 (2026-05-12, CEO)

* 가이드 콘텐츠 `app/guides/[slug]/page.tsx` 다수: "최고 차단성", "최고 수준", "쿠션성 최고" 등.
  * 문맥: 패키징 소재 속성 교육 (특정 vendor 자랑 아님). Legal 자문은 vendor 평가 라벨 금지에 초점 — 교육 콘텐츠 내 소재 속성 비교는 borderline.
* `app/guides/[slug]:4309` 친환경 인쇄 카피: "공식 인증서로 반드시 검증" — 이미 §3 위반 방지 표현.
* design-preview pages: noindex metadata 적용 — production 영향 없음.

## Acceptance

CMO 가 가이드 콘텐츠 + 메인 카피 + 카테고리/회사 페이지 전수 grep:

1. **위험 패턴 grep** (regex 추천): `최저가|최고\b|1위|2위|3위|선정|검증된|공식 (인정|업체|채널 제외)|독점|유일|보장|업계.*1위|업계 표준`
2. **분류**:
   * **vendor 평가 라벨** (e.g., "검증된 업체", "추천 업체") → 즉시 수정 또는 제거.
   * **소재/제품 속성** (e.g., "최고 차단성") → 사실 기준이면 유지 가능, 비교급 ("X 가 Y 보다 차단성 우수") 으로 wording 부드럽게.
   * **vendor 정렬 UI 라벨** (e.g., "추천 정렬") → 사실 라벨로 교체 (등록순/거리순/사양 일치도).
3. 수정 사항을 별도 PR 로 ship.
