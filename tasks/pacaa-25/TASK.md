## 배경

Vision Top-level 목표가 Goals 섹션에 등록됐어. 이를 측정 가능한 Goal
트리로 분해해서 보드 검토를 받아.

Vision 목표 제목 : Packlinx Vision — Irreplaceable B2B Discovery Infrastructure for Korea's Packaging Industry

## 작업 범위

다음 3개 계층의 Goal을 제안해:

1. **Top Goal** 1개 (6개월 시한)
2. **Sub-Goal** 3\~5개 (3개월 시한)
3. **Project Goal** 다수 (1\~3주 시한)

모든 Goal은 부모-자식 관계로 연결돼. Top Goal의 부모는 Vision 노드,
Sub-Goal의 부모는 Top Goal, Project Goal의 부모는 Sub-Goal.

## 시한 단축 가이드 (Packlinx 특수성)

Packlinx는 인간 회사가 아니라 AI 에이전트 회사야. 일반 비즈니스 시한을
그대로 쓰지 마. 영역별로 다른 속도를 가정해

각 Goal의 시한은 그 Goal이 어느 영역에 속하는지에 따라 다르게 잡아.

## 각 Goal에 필수 포함 항목

description 필드에 모두 명시:

* **측정 방법**: 어떤 숫자, 어떤 도구로, 어떤 주기로 측정
* **Baseline**: 현재 값. 측정 불가하면 "측정 시스템 구축"을 별도
  선행 Goal로 분리
* **성공 임계값**: 이 숫자에 도달하면 Goal 달성
* **Kill criterion**: 이 숫자에 도달 못 하면 무엇을 멈추거나 바꿀지
* **Owner**: 어떤 직무가 책임지는가. 현재 조직에 owner가 없으면
  채용 제안서를 별도 PACAA 티켓으로 트리거
* **선행 의존성**: 어떤 Goal이 먼저 충족돼야 시작 가능한가

## Vision 4개 관찰 신호의 측정 정의

Vision 문서에 명시된 4개 신호 각각에 대해 측정 시스템 정의 포함:

1. 자발적 정보 수정 요청 — 어떻게 측정?
2. 자발적 유료 노출 요청 — 어떻게 측정?
3. 신규 업체 등록 우선순위 — 어떻게 측정?
4. 바이어 검색 행동 — 어떻게 측정?

각 신호의 측정 시스템이 부재하면 그 시스템 구축 자체를 Sub-Goal로
분리해.
