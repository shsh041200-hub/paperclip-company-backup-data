## 부모 이슈

## 보드 요구사항 (원문)
> 대시보드 상단에 지금 진행중인 작업이 어떤 작업인지 또 작업이 진행중이 아니라면 왜 진행중이 아니고 무엇을 기다리고 있는거이며 어떤 작업까지 했는지 이모든건 초보자가 본다는 가정하에 진짜 쉬운말로 한국어로 번역해서 알려줘야해

## 작업 정의

**1단계 — 설계 (이 이슈)**

다음을 포함한 설계 문서를 plan document로 첨부:

1. **현재 Paperclip 대시보드 표면 조사.** `/`, `/PACAA`, `/PACAA/agents/<urlkey>` 중 보드가 "대시보드"라고 부를 가능성이 가장 높은 페이지를 찾고 그 페이지 상단의 현재 구조를 캡처/요약. 보드가 매일 가장 많이 보는 페이지가 1순위.
2. **플러그인 vs 코어 수정 선택 + 근거.** `paperclip-create-plugin` 으로 가능하면 플러그인 우선 (역행 가능, 격리). 플러그인 표면이 부족하면 코어 mod 근거 제시.
3. **상태 모델 매핑.** Paperclip API에서 끌어올 수 있는 상태 → 한국어 한 줄 카피로 매핑 표.
   - 진행 중: 어떤 에이전트가 어떤 이슈를 어떤 단계에서 작업 중인지
   - 대기 중 (왜 안 움직임): blocker / interaction pending / approval pending / paused / errored / over-budget — 각각을 "무엇을 기다리는지" 한 줄로 풀어쓰기
   - 완료: 최근 N개 (제안 N + 근거)
4. **"초보자 한국어" 카피 룰.** 합니다체 기본, 영어 jargon 금지 (`in_progress` → `진행 중`, `request_confirmation` → `사장님 승인 기다리는 중` 등). 카피 템플릿 5~10개 초안 + 매핑 케이스별 예시.
5. **레이아웃 와이어프레임.** ASCII 또는 Markdown 표 수준이면 충분. 패널 1개로 끝낼지, "진행 중 / 대기 중 / 최근 완료" 3섹션으로 나눌지 결정.
6. **킬 기준.** 출시 4주 후 보드가 패널을 한 번도 코멘트에서 인용하지 않으면 archive. 측정 방법 1줄.

**2단계 — 보드 승인 게이트**

설계 문서 v1 첨부 후 `request_confirmation` 인터랙션으로 보드 승인 요청. 승인 전까지
