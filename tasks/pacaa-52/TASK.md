## 목적

Paperclip Routines를 활용해 보드(나)의 일상 부담을 줄이고, 회사 운영을
자동화한다. 어떤 Routines를 추가하면 좋을지 분석 후 추천. 이때 기존&#x20;

## 분석 기준

다음 4가지 관점에서 Routines 후보를 평가해 추천:

1. **보드 시간 절약**: 보드가 매번 직접 확인하던 작업을 자동화
2. **에이전트 자가 정렬**: 에이전트가 알아서 자기 상태를 점검/보고
3. **임계값 자동 감지**: Goal 트리의 임계값/Kill criterion 자동 모니터링
4. **학습 누적**: Journal/회고 같은 학습 루프의 자동화

## 산출물 형식

다음 표 형식으로 추천 리스트:

\| 우선순위 | Routine 이름 | 실행 주기 | 담당 에이전트 | 효과 | 구현 난이도 |

각 추천에 대해:

* 이 Routine이 해결하는 문제 한 줄
* 보드 또는 에이전트의 어떤 일상 작업을 대체하는지
* 시작 시점 (지금 즉시 / 페어 1 안정 후 / 페어 2 합류 후)

## 추천 개수

5\~8개 이내. 다 한꺼번에 도입하지 않고 단계 도입 권장.

## 우선순위 가이드

다음 기준으로 우선순위 매겨:

* **즉시 도입**: 보드 시간 절약 효과 큼 + 구현 쉬움
* **2주 후 도입**: 페어 1 안정화 후 가치 발생
* **1개월 후 도입**: 데이터 축적 후 의미 있는 것 (예: 주간 회고)

## 보드 결정 게이트

추천 받은 후 보드가 어떤 것을 즉시 도입할지 선택. 선택된 것만
구현 진행.

## 제약

* 사이드 프로젝트 무게로 처리 (분해 최소화)
* Vision/Goal 직접 기여는 아님. 운영 효율 향상 목적.
* 결재 게이트 1회 (추천 → 보드 선택 → 구현)

완료 시 보드 사인오프 요청.
