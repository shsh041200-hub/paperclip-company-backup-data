## PACAA-16: /products 10개 + /services/printing-design 라우트 신설

**Ticket:** PACAA-16 — 신규 3축 구조 첫/세 번째 축 라우트 신설
**Goal ancestry:** PACAA-15 (업체 카테고리 개선안) → 사용자 발견성 개선

### Options considered

1. **Shared filter component extraction** — Extract common FilterDropdown/MobileFilterSheet into `src/components/filters/` and import across all three routes (categories, products, services).
2. **Route-local pattern replication** — Create dedicated filter components per route, following the existing co-located pattern from `src/app/categories/[slug]/`.

### Benchmark

**Vercel/Next.js App Router convention**: co-locate route-specific client components with their pages. Shared components go in `src/components/` only when 3+ consumers exist with identical interfaces.

### Decision

Option 2 — route-local pattern replication. Each route has different filter requirements:
- Products: material + cert + sample (no form/subtype — page IS the form)
- Services: subtype + cert + sample (no material/form)
- Categories: full set (material + form OR subtype + cert + use-case + sample)

Sharing would require conditional logic inside a generic component, adding complexity for minimal dedup. Two-way door — can extract shared components later if a third product/service type is added.

### Reversibility

Two-way door. Route-local components can be refactored into shared components at any time without schema or API changes.
